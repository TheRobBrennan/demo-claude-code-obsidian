# Building a Second Brain

Tiago Forte's system for using digital notes as an extension of your memory and thinking. The central argument: your mind is for having ideas, not storing them. Offload storage so you can focus on synthesis.

## PARA — the organizational system

Everything in your digital life goes into one of four categories:

- **Projects** — things with a deadline and a goal (active)
- **Areas** — ongoing responsibilities with no end date (career, health, finances)
- **Resources** — reference material you might use someday
- **Archives** — inactive items from the other three categories

The key insight: **organize by actionability, not by topic.** A note about nutrition goes in your Health area, not a Nutrition folder — because when you need it, you're working on health, not browsing a topic.

## The CODE workflow

- **Capture** — get it out of your head immediately
- **Organize** — put it in PARA, don't overthink it
- **Distill** — on subsequent reviews, bold the most important parts (Progressive Summarization)
- **Express** — create something with it

## Progressive Summarization

A technique for making notes more useful over time without re-reading them in full:

1. First pass: capture the source, highlight key passages
2. Second pass: bold the most important highlights
3. Third pass: add an executive summary in your own words

Each layer takes more time but produces higher signal. You only go deeper on notes you actually return to.

## Intermediate packets

One of the more useful ideas in the book: break work into "intermediate packets" — small, reusable chunks that can be recombined across projects. A good note is an intermediate packet. A summary is an intermediate packet. A framework you developed for one client can be an intermediate packet for the next.

## Critique / tension with Zettelkasten

PARA organizes by project/area. Zettelkasten organizes by idea. They're not incompatible — many people use PARA for active work and a more link-heavy system for reference knowledge — but they reflect different assumptions about how knowledge gets used.

PARA is better for people who work on discrete projects. Zettelkasten is better for people who think across long time horizons and domains.

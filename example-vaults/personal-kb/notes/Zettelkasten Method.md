# Zettelkasten Method

The Zettelkasten (German: "slip box") is a note-taking system developed by sociologist Niklas Luhmann, who used it to write more than 70 books and 400 academic articles over his career. The core idea is that notes should be linked to each other rather than filed in hierarchical folders.

## Core principles

**Atomic notes.** Each note contains exactly one idea. If you find yourself using "and" in a note title, split it.

**Permanent notes.** After capturing a fleeting idea, rewrite it in your own words as if writing for a future reader (even if that reader is only you). This forces understanding.

**Linking over filing.** Instead of deciding which folder a note "belongs" to, ask which existing notes it connects to. A note that links to nothing is a note you'll never find again.

**Bottom-up structure.** Topics and outlines emerge from the connections between notes — you don't impose structure from the top down.

## The three note types

1. **Fleeting notes** — quick captures, meant to be processed within a day or two. These can be rough.
2. **Literature notes** — what you took from a source, in your own words. One per source.
3. **Permanent notes** — the actual Zettelkasten. One idea, linked to related permanent notes.

## What makes a good link?

A link should answer the question: "under what circumstances would I want to come across this note while thinking about that note?" Not "are these two topics related?" but "is one useful when thinking about the other?"

## Common mistakes

- Collecting notes instead of connecting them
- Filing by topic instead of linking by idea
- Waiting until you "understand" something to write it down
- Making notes too long (violates atomicity)

## Why it works

Knowledge compounds when ideas collide unexpectedly. The Zettelkasten creates those collisions by forcing you to place new ideas next to old ones. Luhmann described it as having a conversation with a surprisingly well-read, occasionally annoying second brain.

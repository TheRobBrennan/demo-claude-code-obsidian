# Obsidian vs Notion

My working comparison for personal knowledge management. Not trying to be comprehensive — just the things that matter for how I use these tools.

## Core difference in philosophy

**Notion** is a workspace: collaborative, structured, database-first. Good for teams, projects, and anything where you want tables, views, and sharing.

**Obsidian** is a thinking tool: local-first, markdown, link-first. Good for personal knowledge, writing, and anything where longevity and privacy matter.

They're not really competing for the same use case if you think about it clearly.

## Where Notion wins

- Databases and filtered views (kanban boards, calendars, tables with custom properties)
- Collaboration and sharing
- Embedding structured data (forms, relations between tables)
- Web clipper that actually works well
- No setup required — just open a browser

## Where Obsidian wins

- Local-first: files are markdown on disk, no vendor lock-in
- Graph view and backlinks — seeing the actual connections between ideas
- Speed (local app, no round-trips to a server)
- Plugin ecosystem is extraordinary — custom workflows for almost any use case
- Works offline completely
- Privacy: your notes don't live on someone else's server
- Longevity: markdown files will be readable in 30 years. Notion exports are messy.

## The longevity argument

This is the one that settled it for me. Notes I write today should be readable in 10–20 years without depending on any particular company surviving. Plain markdown files in a folder satisfy this. Notion's proprietary format does not.

## What I actually use each for

- **Obsidian:** permanent notes, reading notes, project thinking, daily notes, writing drafts
- **Notion:** shared project boards with collaborators, client-facing documents, anything that needs a table or a view

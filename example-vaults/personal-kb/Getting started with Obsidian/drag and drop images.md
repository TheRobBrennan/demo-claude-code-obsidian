## The one plugin worth knowing

**[[Image Toolkit]]** (community plugin) adds click-to-zoom and pan — useful if you're dropping in dense screenshots you want to examine closely.

Bottom line: drag and drop works great for capturing reference screenshots. Where Obsidian falls short is _layout control_ — if you need images to be a designed part of the note (captions, side-by-side, resize handles), you're fighting the markdown model. For reference/documentation screenshots it's totally fine.
